# Migrating to v0.13: selection and folder-structure flags

v0.13 reshapes how you tell kei *what* to download and *where* to put it. Existing configs keep working with one warning cycle. Old flag names go away in v0.20.

## What you actually need to do

- If you use `--exclude-album`, switch to `--album '!NAME'`.
- If you have `{album}` in `--folder-structure`, kei auto-migrates it to `--folder-structure-albums` and prints a warning. Optional: update your CLI/TOML to silence the warning.
- If you run `kei sync` with no flags and you have many albums, expect more API calls per sync. Output paths stay the same when your `--folder-structure` doesn't include `{album}`.

Everything else is additive: new flags exist, old flags keep working until v0.20.

## Quick before/after

```sh
# Before (v0.12)
kei sync -a Vacation -a Family --exclude-album Drafts --folder-structure '{album}/%Y/%m/%d'

# After (v0.13, equivalent)
kei sync --album Vacation --album Family --album '!Drafts' --folder-structure-albums '{album}/%Y/%m/%d'
```

```sh
# New: separate template for unfiled photos (closes #215)
kei sync --folder-structure-albums '{album}' --folder-structure '%Y/%m'
# Albumed photos -> Vacation/IMG.JPG
# Unfiled photos -> 2024/06/IMG.JPG
```

## Selection flags

| Flag | Values | Default | Notes |
|---|---|---|---|
| `--album NAME` | literal name, `all`, `none`, `!literal` | `all` | Repeatable. `all` is every user album. |
| `--smart-folder NAME` | literal name, `all`, `all-with-sensitive`, `none`, `!literal` | `none` | Repeatable. `all` excludes Hidden + Recently Deleted; `all-with-sensitive` includes them. |
| `--library NAME` | raw zone name (`PrimarySync`, `SharedSync-...`), truncated 8-char form, `all`, `none`, `primary`, `shared`, `!literal` | `primary` | Repeatable. |
| `--unfiled` | `true`, `false` | `true` | Boolean. Bare `--unfiled` sets true; `--unfiled false` disables. |

Defaults apply per category. Setting `--smart-folder Favorites` doesn't change the `--album` default; you still get every album plus Favorites.

`!` prefix means exclude. Bare `--album '!Family'` means "every album except Family" (the `!` operates on the category default, which is `all`).

## Layout flags

| Flag | Default | Tokens accepted |
|---|---|---|
| `--folder-structure TEMPLATE` | `%Y/%m/%d` | strftime, `{library}` |
| `--folder-structure-albums TEMPLATE` | `{album}` | strftime, `{library}`, `{album}` |
| `--folder-structure-smart-folders TEMPLATE` | `{smart-folder}` | strftime, `{library}`, `{smart-folder}` |

`--folder-structure` is the unfiled-pass template. The two category templates default to flat per-folder layouts (`{album}` / `{smart-folder}`) because albums are already organised collections. Add date hierarchy explicitly: `--folder-structure-albums '{album}/%Y/%m/%d'`.

## Tokens

| Token | Allowed in | Placement | Renders as |
|---|---|---|---|
| `{library}` | every template | first segment, single occurrence | `PrimarySync` or `SharedSync-A1B2C3D4` (truncated 8-char UUID prefix) |
| `{album}` | `--folder-structure-albums` only | first segment, or second after `{library}` | album name |
| `{smart-folder}` | `--folder-structure-smart-folders` only | first segment, or second after `{library}` | smart-folder name |
| strftime (`%Y`, `%m`, `%B`, `%d`, ...) | every template | anywhere | as today |

Tokens used in the wrong template bail at startup with a clear error.

## Multi-library disambiguation

`--library` now takes multiple values. With more than one library in scope, put `{library}` in your templates to keep paths separate:

```sh
kei sync --library all --folder-structure-albums '{library}/{album}'
# Photos -> PrimarySync/Vacation/IMG.JPG
#       and SharedSync-A1B2C3D4/Vacation/IMG.JPG
```

Without `{library}` in the template, kei warns at startup that paths will share a namespace and the state DB will silently dedup cross-library matches. The warning is informational; the run continues.

## TOML mapping

`[filters]` extends with per-category arrays. `[download]` extends with per-category templates.

```toml
[filters]
albums = ["all", "!Family"]      # CLI: --album all --album '!Family'
smart_folders = []               # CLI: (no --smart-folder flags)
libraries = ["primary"]          # CLI: --library primary
unfiled = true                   # CLI: --unfiled

[download]
folder_structure = "%Y/%m/%d"
folder_structure_albums = "{album}"           # flat per-album default
folder_structure_smart_folders = "{smart-folder}"
```

Old keys keep working through v0.19 with a warning, removed in v0.20:

| Old key | New key |
|---|---|
| `[filters].album` | `[filters].albums` |
| `[filters].exclude_albums` | merge into `[filters].albums` as `"!name"` entries |
| `[filters].library` (single string) | `[filters].libraries` (array) |

## Removed / renamed in v0.20

| Old | Replacement | Status |
|---|---|---|
| `--exclude-album NAME` | `--album '!NAME'` | warns now, removed in v0.20 |
| `{album}` in `--folder-structure` | `--folder-structure-albums "{album}/..."` | auto-migrates with warning, removal in v0.20 |
| Auto-`-a all` from `{album}` token in template | covered by new default `--album all` | removed in v0.20 |
| `[filters].album` (TOML) | `[filters].albums` | warns now, removed in v0.20 |
| `[filters].exclude_albums` (TOML) | `[filters].albums` with `!` entries | warns now, removed in v0.20 |
| `[filters].library` (TOML, single string) | `[filters].libraries` (array) | warns now, removed in v0.20 |

## Behavior changes worth knowing

- **`kei sync` with no flags** now enumerates each user album and runs an unfiled pass instead of one library-wide stream. More API calls per sync. On-disk paths stay identical to v0.12 only when your `--folder-structure` has no `{album}` token. With the default template (`%Y/%m/%d`), output is unchanged.
- **`--unfiled` defaults to `true` regardless of `--album`.** `kei sync --album Vacation` runs the Vacation pass *and* the unfiled pass. To restrict to just Vacation, add `--unfiled false`.
- **`--smart-folder all` skips Hidden + Recently Deleted** (mirroring iOS Photos). Use `--smart-folder all-with-sensitive` for literal everything.
- **Album in multiple folders gets one copy per album folder** (unchanged from v0.12). Photos in any album are excluded from the unfiled pass.
- **An album literally named `all`, `none`, or starting with `!`** is unaddressable. Documented limitation; corner case.

## State DB schema v8

The `assets` table primary key changes from `(id, version_size)` to `(library, id, version_size)` so the same asset can track distinct rows across PrimarySync and shared libraries. Migration runs automatically on first sync after upgrade and backfills existing rows to `library = "PrimarySync"`. Single-library users see no behavior change. Users who were already syncing shared libraries pre-v8 may have a small set of rows misattributed to PrimarySync from the backfill; the next sync writes correct per-zone rows alongside, and the misattributed rows become harmless leftovers (they don't trigger re-downloads -- the live asset is keyed correctly).

## Validation

| You did this | kei does this |
|---|---|
| `--album Vacatiom` (typo, no such album) | Bails at startup with "Album 'Vacatiom' not found. Did you mean 'Vacation'?" |
| `--album '!Vacatiom'` (typo in exclusion) | Warns, continues. Excluding nothing is harmless. |
| `--album Foo --album '!Foo'` | Bails at parse time. |
| `--library none` (no source selected) | Bails. |
| `--album none --smart-folder none --unfiled false` (everything turned off, valid library) | Runs as no-op, exits 0. |
| `{album}` in `--folder-structure-smart-folders` | Bails at startup. |
